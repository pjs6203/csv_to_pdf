첨부한 PDF는 영어 단문독해 문제집 스캔본이다. 이 PDF를 내가 제공한 `csv_to_pdf.py` 프로그램에서 바로 사용할 수 있는 CSV로 변환해라.

중요:
이 CSV는 일반적인 "문항별 1행" 구조가 아니라, 반드시 "한 행 = 한 지문 묶음" 구조여야 한다.
단문독해는 보통 한 지문에 한 문항이 붙어 있으므로 대부분 `001-001`, `002-002`처럼 한 문항짜리 qrange가 된다.
만약 하나의 지문에 여러 문항이 붙어 있으면 한 행만 만들고, 모든 문항을 `questions_json` 컬럼 안에 JSON 배열로 넣어라.

최종 CSV 컬럼은 아래 7개만 사용한다.

qrange,chapter,source_pages,passage,questions_json,underlines_json,notes

다른 컬럼은 만들지 마라.

각 컬럼 작성 규칙은 다음과 같다.

1. qrange
- 지문 묶음 번호를 넣는다.
- 한 문항짜리 지문도 반드시 `001-001`, `002-002`, `013-013`처럼 3자리 숫자-3자리 숫자 형식으로 쓴다.
- 여러 문항이 한 지문에 붙어 있으면 `014-016`처럼 시작 번호와 끝 번호를 하이픈 `-`으로 연결한다.
- 물결표 `~`는 사용하지 마라.

2. chapter
- 해당 지문이 속한 챕터명을 넣는다.
- 챕터 표지, 상단 헤더, 목차를 참고해서 원본 표기와 최대한 맞춘다.
- 형식은 다음처럼 통일한다.
  - `CHAPTER 01 챕터명`
  - `CHAPTER 02 챕터명`
- 챕터명이 OCR로 불확실하면 가능한 범위에서 복원하고 notes에 기록한다.

3. source_pages
- 원본 PDF에서 해당 지문과 문항이 등장한 페이지 번호를 적는다.
- 지문과 문항이 같은 페이지면 `9`처럼 적는다.
- 지문과 문항이 다른 페이지에 걸치면 `"7,8"`처럼 적는다.
- CSV에서 쉼표가 포함되면 반드시 큰따옴표로 감싼다.
- 인쇄된 페이지 번호가 보이면 인쇄된 페이지 번호를 우선 사용한다.
- 인쇄된 페이지 번호가 없거나 불명확하면 PDF 페이지 번호를 사용하고 notes에 `source page uncertain`이라고 적는다.

4. passage
- 해당 qrange의 영어 지문 전체를 넣는다.
- 지문 안의 줄바꿈은 제거하고, 자연스러운 공백 하나로 합친다.
- 문제 번호 `001.` 같은 표기와 지문 위 번호는 passage에 넣지 않는다.
- OCR 오류를 그대로 두지 말고 원본 이미지를 보고 가능한 정확히 복원한다.
- 빈칸은 반드시 `______`로 통일한다.
- 밑줄 단어는 passage 안에서는 일반 텍스트로 넣고, 별도로 `underlines_json`에 기록한다.
- 상단 헤더, 하단 페이지 번호, 출판사명, 장식 문구, 스캔 노이즈는 passage에 넣지 않는다.
- 지문이 짧은 단문이어도 passage는 비워 두면 안 된다.

5. questions_json
- 해당 qrange에 속한 모든 문항을 JSON 배열로 넣는다.
- 단문독해는 보통 한 행에 문항 객체 하나만 들어간다.
- 반드시 유효한 JSON이어야 한다.
- 각 문항 객체의 구조는 반드시 아래와 같다.

[
  {
    "num": "001",
    "stem": "Which of the following is correct according to the passage?",
    "options": {
      "A": "선택지 A 내용",
      "B": "선택지 B 내용",
      "C": "선택지 C 내용",
      "D": "선택지 D 내용",
      "E": "선택지 E 내용"
    }
  }
]

- `num`은 반드시 3자리 문자열로 쓴다. 예: `"001"`, `"014"`, `"120"`
- `stem`에는 문제 발문만 넣는다.
- 문제 번호 `001.` 같은 표기는 stem에 넣지 않는다.
- 선택지에는 `(A)`, `(B)`, `(C)`, `(D)`, `(E)` 또는 `①`, `②`, `③`, `④`, `⑤` 기호를 넣지 말고 내용만 넣는다.
- 원본 선택지가 `①`부터 `⑤`까지라면 `①=A`, `②=B`, `③=C`, `④=D`, `⑤=E`로 변환한다.
- 선택지는 반드시 A, B, C, D, E 순서로 정리한다. 원본에 A-D만 있으면 A-D만 넣는다.
- 선택지가 두 열로 배치되어 있어도 원본 번호 순서에 맞게 정리한다.
- 선택지가 줄바꿈되어 있으면 하나의 문자열로 합친다.
- JSON 안의 따옴표는 CSV 규칙에 맞게 안전하게 이스케이프한다.
- 가장 안전한 방식은 Python `csv.DictWriter`와 `json.dumps(..., ensure_ascii=False)`를 사용해 CSV를 생성하는 것이다.

6. underlines_json
- 지문이나 문제에서 밑줄 친 단어/구가 있으면 JSON 배열로 넣는다.
- 예: `["telling"]`
- 여러 개면 `["word1", "word2"]`
- 없으면 빈 배열 `[]`
- `underlines_json`에 넣은 표현은 passage, stem, options 안의 실제 텍스트와 철자가 정확히 일치해야 한다.
- 확실하지 않으면 notes에 `underline uncertain`이라고 기록한다.

7. notes
- OCR 의심, 페이지 분리, 선택지 일부 불명확, 번호 누락 가능성, 챕터명 불확실 등을 기록한다.
- 문제가 없으면 빈칸으로 둔다.
- 예:
  - `OCR uncertain: word after however`
  - `question split across pages`
  - `source page uncertain`
  - `underline uncertain`

추출 제외 대상:
- 표지
- 머리말
- 목차
- 챕터 표지
- PREVIEW / EXPLANATION 장식 문구
- 상단 헤더
- 하단 페이지 번호
- 출판사 로고
- 스캔 노이즈
- 의미 없는 OCR 잔재
- 해설 페이지가 있으면 해설은 제외하고 문제와 지문만 추출한다.

연결 규칙:
- 단문독해는 일반적으로 각 번호가 독립된 지문과 문항이다.
- 번호 `001`의 지문과 문항은 한 행으로 만들고 qrange는 `001-001`로 쓴다.
- 번호 `002`의 지문과 문항은 한 행으로 만들고 qrange는 `002-002`로 쓴다.
- 만약 한 지문에 `009`, `010`, `011` 세 문항이 붙어 있으면 qrange는 `009-011`로 쓰고, 세 문항을 모두 같은 행의 `questions_json`에 넣는다.
- 지문과 문항이 서로 다른 페이지에 있어도 번호를 기준으로 반드시 연결한다.
- `qrange`가 `009-011`이면 `questions_json` 안의 문항 번호는 정확히 009, 010, 011이어야 한다.

검수 규칙:
- 모든 qrange는 `^\d{3}-\d{3}$` 형식이어야 한다.
- qrange의 시작 번호부터 끝 번호까지 모든 문항이 questions_json 안에 정확히 들어 있어야 한다.
- question number가 중복되면 안 된다.
- passage가 비어 있으면 안 된다.
- 각 문항에는 최소 2개 이상의 선택지가 있어야 하고, 원본이 5지선다면 A-E 5개가 모두 있어야 한다.
- OCR로 `conducted`가 깨지거나 `B니t`, `yo니ng`, `vahje`처럼 한글/깨진 문자가 섞인 경우, 이미지 원본을 보고 정상 영어로 고친다.
- 확신이 없으면 임의 추측하지 말고 notes에 기록한다.
- `validate_csv.py`를 실행했을 때 오류가 없어야 한다.

출력 방식:
- UTF-8 CSV 파일로 제공한다.
- CSV 첫 줄은 반드시 아래 헤더여야 한다.

qrange,chapter,source_pages,passage,questions_json,underlines_json,notes

- 마크다운 표로 출력하지 마라.
- 설명문을 CSV 안에 섞지 마라.
- 최종 CSV는 `csv_to_pdf.py`와 `validate_csv.py`에서 바로 사용할 수 있어야 한다.

먼저 전체 PDF를 한 번에 변환하지 말고, 001-013번까지만 샘플 CSV로 만들어라.
내가 승인하면 같은 규칙으로 전체 PDF를 끝까지 변환해라.
